重构 useHooks

When we started working on react.gg, a primary focus of ours was figuring out how to get students comfortable working with "production" level React code, without the burden or context of needing to dive into a fully fledged project.

I'm sure you've experienced it before. In an attempt to make the course "hands-on", the instructor has you clone a starter project from Github, npm install two dozen packages, and then you spend the next 14 hours watching them glue it together.

This is the “你不可能靠读书学会骑自行车” approach to developer education. That's fantastic advice, if you're a 4 year old. In reality, professional cyclists do learn all about riding, both the physical act and how to most efficiently train, from reading.

I know this seems controversial, but you would never tell a professional basketball player to “去玩就完了” – so why is "just build things" such a common trope when it comes to learning technical topics?

My running theory is it's because it's the most generic, widely applicable advice that is certain to be true. You do get better by “动手去做”. The problem is the most widely applicable approach is almost never the most efficient one.

如果说职业运动员的训练方式能给我们什么启发，那就是他们精通于把比赛中可能遇到的场景在训练中复现出来——没有对手、没有比分、没有观众，只有对技艺极度专注的打磨。

When we built react.gg, this is the experience we wanted to recreate.

到目前为止，你已经在完成的挑战中经历了 45 次这种真实场景的模拟。它们让你能熟悉 React 的方方面面，而无需一头扎进一个完整的项目。没有 repo、没有 node_modules、没有繁琐的上下文——只有一个让你专心打磨技艺的高度聚焦环境。

The tradeoff here, of course, is that a "real-world simulated environment" is still just that – simulated.

The course was still, despite our best effort, missing something that would allow you to get comfortable with the actual code you'll see in the actual projects you'll work on.

Then we had an idea. If we wanted you to get real-world experience, what better way than to have you build a real-world React library. Naturally, this led us to the creation of useHooks - a library of 50 modern, real-world, production ready React hooks that as of this writing gets more than 20,000 downloads a week.

What I love about useHooks, and what's impossible to do with traditional course projects, is that everything we've covered in the course so far is used in the library. There isn't one thing you've learned that you won't see again in one form or another in useHooks. Not only that, but by the nature of building reusable hooks, you'll also get more familiar with fundamental browser APIs like localStorage, matchMedia, and IntersectionObserver to name a few.

最棒的是，你可以在自己的项目中直接使用 useHooks（或者你自己实现的版本）。

祝你好运！
