Reality Check

Up until this point, we've seen how beautifully simple React is in its purest form. You encapsulate UI, state, and event handlers into components, compose those components together, passing data down the component tree as needed via props.

The problem, of course, is nothing muddies the simplicity of a mental model quite like the complexities of reality.

In the real world, it's not uncommon to need to do things outside of React. Fetching data from a server, interacting directly with the DOM, or even using native browser APIs like setTimeout or localStorage are all examples of real world use cases that don't fit neatly into the React's model.

For React to be of any practical use and not just a theoretical tool we learn about in Computer Science programs, it needs to handle these outside world use cases – and do so in a way that doesn't completely destroy the simplicity of the mental model.

That's what this section is dedicated to, specifically learning how get React to play nicely with the outside world.

Let's dive in.
